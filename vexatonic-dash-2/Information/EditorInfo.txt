Editor 기획

기본적으로, Lane을 먼저 찍은 뒤에 Lane 위에 Note들을 찍어야 함.

-Lane 입력-
기본적으로 2개의 상태가 존재.
Ready: 아직 Lane 찍기를 시작하지 않은 단계.
Placing: Ready 이후 단계.

-Ready 단계에서의 case-
Case 1. 비어 있는 곳에 Lane을 찍으려 한다면 x=0인 지점에 preview가 생기며, 해당 레인은 자동으로 Initial lane이 됨.
Case 2. 레인 위에 마우스를 올려놓고 Lane을 찍으려 한다면 해당 레인에 preview가 생기며, 해당 지점부터 lane이 분기가 됨.
Case 3. 레인의 끝에서 Lane을 찍으려 한다면 해당 지점에 preview가 생기며, 그 지점은 lane의 keyframe(관절)이 됨.
1, 2, 3 셋 중 하나의 단계를 거치면 Ready에서 Placing으로 이동.

-Placing 단계에서의 case-
1, 2, 3 경우 모두 lane의 시작점 내지는 관절이 저장되었을 것이며, 해당 지점에서 마우스로 다음 관절까지 이동시킬 수 있음.
마우스 재클릭 시 해당 lane의 새로운 관절이 저장됨.
추가로 Placing 단계에서 Shift 버튼을 누르고 있다면, x좌표가 일정한 레인을 찍을 수 있음.

-Trigger 찍기-
1. Ready 단계: Trigger의 위치를 정한다.
2. Ready에서 마우스 클릭한 경우: Trigger의 위치를 확정한다. EditorTrigger 객체를 하나 생성해 해당 Trigger 노드를 할당한다.
   단노트와 동일하게, Placing 단계는 없다. 처음 생성 시 모든 값이 0으로, 이 경우 트리거는 아무것도 하지 않는다.

-Modify Trigger-
1. Ready 단계: 어떤 트리거를 선택할 지 음영으로 표시한다.
2. Placing 단계: 클릭하면 UI가 등장. 다른 에디터 조작을 할 수 없다. UI를 닫으면 Ready 단계로 돌아간다.
